I assumed I got the raw data and did the following analysis to answer the question

1. Analysis of raw data¶

Analysis pipeline Ref:Comprehensive Molecular Characterization of Urothelial Bladder Carcinoma)

1.1 DNA Sequencing analysis¶

(1)Quality control of raw data

(2)Mapping raw reads to GRCh37 by BWA

(3)Picard used to make a single bam stores all reads and calibrated qualities along with their alignments to the genome

(4)Variant calling and remove library preparation artifact

Several pipeline is provided such as Firehose. Firehose include following steps:

  1. Quality control – confirms identity of individual tumor and normal to avoid mix-ups between tumor and normal data for the same individual.
  2. Local realignment of reads – realigns sites potentially harboring small insertions or deletions in either the tumor or the matched normal to decrease the number of false positive single nucleotide variations caused by misaligned reads.
  3. Identification of somatic single nucleotide variations (SSNVs) – Mutect algorithm – candidate SSNVs were detected using a statistical analysis of the bases and qualities in the tumor and normal BAMs.
  4. Identification of somatic small insertions and deletions – Indelocator algorithm – putative somatic events were first identified within the tumor BAM file and then filtered out using the corresponding normal data.

1.2 Mutation significance analysis¶

The statistical significance of mutation frequency in each gene was determined using the algorithm MutSig. Use qvalue as cutoff and (AF) >= 0.05 to remove potential subclonal mutations. Include genes significantly mutated genes from COSMIC database

1.3 How to improve accuracy of analysis.¶

From the Ref:A comprehensive assessment of somatic mutation detection in cancer using whole-genome sequencing

We should know:

  • About the library depth(a minimum of 30 × sequence coverage, 100 X sequence coverage would be the best)
  • Effect of tumor purity on mutation calling.(no more than 17% 'contamination' of normal samples in tumour would be great)
  • Effect of tumor to normal sequencing depth ratio(Increasing depth of normal samples is help to decrease the false positive)
  • Effect of somatic mutation calling pipelines. MB.Q and MB.J are the best pipeline.(with the highest F1 score)
  • Effect of caller on mutation calling. According precision, Strelka(up to 0.99) is higher than MuTect(up to 0.87).
  • Effect of primary mutation caller on mutation calling
  • Improvement of pipelines using the benchmark data set

2.3.2 Data analysis¶

From this paper, we know how to analysis the data. Basically, it can be conclude as:

  • We mapped chip-Seq reads on hg19 reference.
  • The mutation densities is determined for all possible types of mutations by counting different types of mutations in 1 Mb windows and normalizing for the sequence composition of each window.( Example, In 1 Mb region , Number of G->A mutation is N1, number of G in this 1 Mb region is N2. So N1/N2 is the mutation density of 'G->A' in this region)
  • For histone modifications, we calculated RPKM values of each histonme for 1 Mb windows along the genome from chip-seq data.
  • We calculated GC content for 1 Mb windows along the genome.
  • We calculated the average number of DNaseI hypersensitivity peaks overlapping 1 Mb windows.
